03
🔌 Method 1 — Connect Using MySQLi

MySQLi (MySQL Improved) is the procedural/OOP extension built into PHP. Fastest and simplest for MySQL-only projects.

PHP — MySQLi Procedural 
<?php
// ── Database credentials ──
$host   = "localhost";
$user   = "root";
$pass   = "";         // XAMPP default: empty
$dbname = "testdb";

// ── Create connection ──
$conn = mysqli_connect($host, $user, $pass, $dbname);

// ── Check connection ──
if (!$conn) {
    die("❌ Connection failed: " . mysqli_connect_error());
}

echo "✅ Connected successfully to MySQL!";

// ── Close connection ──
mysqli_close($conn);
?>
✅ Connected successfully to MySQL!
❌ Connection failed: Access denied for user 'root'@'localhost'
04
🎯 Method 2 — Connect Using MySQLi OOP

Object-Oriented approach is cleaner and preferred for modern PHP development.

PHP — MySQLi Object-Oriented 
<?php
$conn = new mysqli("localhost", "root", "", "testdb");

if ($conn->connect_error) {
    die("❌ Failed: " . $conn->connect_error);
}

echo "✅ OOP Connected to MySQL!";
$conn->close();
?>
05
🛡️ Method 3 — Connect Using PDO (Recommended)

PDO (PHP Data Objects) is the most secure and flexible method. It supports multiple database drivers and prepared statements natively.

PHP — PDO MySQL 
<?php
$host   = "localhost";
$dbname = "testdb";
$user   = "root";
$pass   = "";

try {
    $dsn  = "mysql:host=$host;dbname=$dbname;charset=utf8mb4";
    $pdo  = new PDO($dsn, $user, $pass);

    // Set error mode to exception
    $pdo->setAttribute(PDO::ATTR_ERRMODE,
                     PDO::ERRMODE_EXCEPTION);

    echo "✅ PDO Connected to MySQL successfully!";

} catch (PDOException $e) {
    echo "❌ Connection failed: " . $e->getMessage();
}
?>
💡 PDO supports MySQL, SQL Server, PostgreSQL, SQLite and more — switch databases by just changing the DSN string.

🚀 Run a Query After Connection

07
🔌 Method 1 — Connect Using sqlsrv_connect()

The native Microsoft sqlsrv extension provides sqlsrv_connect() for direct SQL Server connections.

PHP — sqlsrv Extension 
<?php
// ── Server and credentials ──
$serverName = "LPT5032";       // or ".\SQLEXPRESS"
$connOptions = [
    "Database"  => "aplbanking",
    "Uid"       => "sa",
    "PWD"       => "Bitopi@1234"
];

// ── Attempt connection ──
$conn = sqlsrv_connect($serverName, $connOptions);

if ($conn) {
    echo "✅ Connected to SQL Server successfully!";
    sqlsrv_close($conn);
} else {
    echo "❌ Connection FAILED!<br>";
    foreach (sqlsrv_errors() as $err) {
        echo "Code: "    . $err['code']    . "<br>";
        echo "Message: " . $err['message'] . "<br>";
    }
}
?>
✅ Connected to SQL Server successfully!

🛡️ Method 2 — Connect Using PDO (Recommended)

🚀 Run a Query After Connection

Execute a SQL SELECT query against SQL Server and display the results.

PHP — sqlsrv Query 
<?php
$conn = sqlsrv_connect("LPT5032", [
    "Database" => "aplbanking",
    "Uid"      => "sa",
    "PWD"      => "Bitopi@1234"
]);

$sql    = "SELECT TOP 10 * FROM dbo.Accounts";
$result = sqlsrv_query($conn, $sql);

while ($row = sqlsrv_fetch_array($result, SQLSRV_FETCH_ASSOC)) {
    echo $row['AccountNo'] . " — " .
         $row['AccountName'] . "<br>";
}
sqlsrv_free_stmt($result);
sqlsrv_close($conn);
?>

🔌 Method 1 — ADO.NET Connection (MySqlConnection)

C# — ADO.NET MySqlConnection 
using MySql.Data.MySqlClient;

class Program
{
    static void Main()
    {
        // ── Connection string ──
        string connStr =
            "Server=localhost;"  +
            "Database=testdb;"  +
            "Uid=root;"         +
            "Pwd=;";             // XAMPP: empty password

        {
            conn.Open();

            Console.WriteLine("❌ Error: " + ex.Message);

    }
}

04
🚀 Run a SELECT Query with ADO.NET

After connecting, use MySqlCommand and MySqlDataReader to execute and read query results.

C# — Execute Query 
using MySql.Data.MySqlClient;

string connStr = "Server=localhost;Database=testdb;Uid=root;Pwd=;";

using var conn = new MySqlConnection(connStr);
conn.Open();

string sql = "SELECT id, name, email FROM users";
using var cmd    = new MySqlCommand(sql, conn);
using var reader = cmd.ExecuteReader();

while (reader.Read())
{
    Console.WriteLine(
        $"ID: {reader["id"]}  Name: {reader["name"]}  Email: {reader["email"]}"
    );
}

// Parameterized INSERT (prevents SQL injection)
string insert = "INSERT INTO users(name,email) VALUES(@name,@email)";
using var ins = new MySqlCommand(insert, conn);
ins.Parameters.AddWithValue("@name",  "Johirul Islam");
ins.Parameters.AddWithValue("@email", "johir@bitopi.com");
ins.ExecuteNonQuery();
Console.WriteLine("✅ Row inserted!");
05
🌿 Method 2 — Entity Framework Core (Code-First)

EF Core is the modern ORM approach. Define your models as C# classes and let EF Core handle SQL automatically.

C# — EF Core with MySQL (Pomelo) 
// 1. Install packages first:
// Install-Package Pomelo.EntityFrameworkCore.MySql
// Install-Package Microsoft.EntityFrameworkCore.Tools

using Microsoft.EntityFrameworkCore;

// ── Model ──
public class User
{
    public int    Id    { get; set; }
    public string Name  { get; set; }
    public string Email { get; set; }
}

// ── DbContext ──
public class AppDbContext : DbContext
{
    public DbSet<User> Users { get; set; }

    protected override void OnConfiguring(DbContextOptionsBuilder opt)
        => opt.UseMySql(
            "Server=localhost;Database=testdb;Uid=root;Pwd=;",
            ServerVersion.AutoDetect(
                "Server=localhost;Database=testdb;Uid=root;Pwd=;")
        );
}

// ── Usage ──
using var db = new AppDbContext();
var users = db.Users.ToList();
users.ForEach(u => Console.WriteLine($"{u.Name} — {u.Email}"));
ℹ️ Run Add-Migration InitialCreate then Update-Database in Package Manager Console to create the database tables automatically.

The connection string is the most critical part. It tells SqlConnection where the server is, which database to use, and how to authenticate.

C# — Connection String Formats

string connStr =
    "Server=LPT5032;"        +  // Server name or IP
    "Database=aplbanking;"   +  // Database name
    "User Id=sa;"            +  // SQL login
    "Password=Bitopi@1234;" +  // SQL password
    "TrustServerCertificate=True;";

// ── Windows Authentication (no password needed) ──

    "Server=LPT5032;"        +

    "Integrated Security=True;" +
    "TrustServerCertificate=True;";

// ── Named Instance ──

// ── With port ──

04
🚀 Execute Queries with SqlCommand

Use SqlCommand to run SELECT, INSERT, UPDATE, DELETE. Always use parameterized queries to prevent SQL injection.

C# — SqlCommand + SqlDataReader 
using Microsoft.Data.SqlClient;

string connStr = "Server=LPT5032;Database=aplbanking;User Id=sa;Password=Bitopi@1234;TrustServerCertificate=True;";

using var conn = new SqlConnection(connStr);
conn.Open();

// ── SELECT ──
string sql = "SELECT TOP 5 AccountNo, AccountName FROM dbo.Accounts";
using var cmd    = new SqlCommand(sql, conn);
using var reader = cmd.ExecuteReader();

while (reader.Read())
{
    Console.WriteLine(
        $"{reader["AccountNo"]} — {reader["AccountName"]}"
    );
}
reader.Close();

// ── INSERT with parameters (safe!) ──
string ins = "INSERT INTO Accounts(AccountNo, AccountName) VALUES(@no, @name)";
using var insCmd = new SqlCommand(ins, conn);
insCmd.Parameters.AddWithValue("@no",   "ACC-001");
insCmd.Parameters.AddWithValue("@name", "Johirul Islam");
int rows = insCmd.ExecuteNonQuery();
Console.WriteLine($"{rows} row(s) inserted.");

🌿 Method 2 — Entity Framework Core (Code-First)

EF Core with SQL Server is the modern ORM approach for .NET — model your database as C# classes, write LINQ queries instead of raw SQL.

C# — EF Core + SQL Server 
// Install packages:
// Install-Package Microsoft.EntityFrameworkCore.SqlServer
// Install-Package Microsoft.EntityFrameworkCore.Tools

using Microsoft.EntityFrameworkCore;

// ── Model ──
public class Account
{
    public int    Id          { get; set; }
    public string AccountNo   { get; set; }
    public string AccountName { get; set; }
}

// ── DbContext ──
public class BankContext : DbContext
{
    public DbSet<Account> Accounts { get; set; }

    protected override void OnConfiguring(DbContextOptionsBuilder opt)
        => opt.UseSqlServer(
            "Server=LPT5032;Database=aplbanking;" +
            "User Id=sa;Password=Bitopi@1234;"    +
            "TrustServerCertificate=True;");
}

// ── LINQ Query Usage ──
using var db = new BankContext();

// Fetch all accounts
var accounts = db.Accounts
    .Where(a => a.AccountName.Contains("Johirul"))
    .ToList();

accounts.ForEach(a =>
    Console.WriteLine($"{a.AccountNo}: {a.AccountName}"));
ℹ️ Run Add-Migration InitDb and Update-Database in Package Manager Console to auto-create SQL Server tables from your C# models.
06
⚙️ Enable SQL Server for Remote/Network Access

If Visual Studio is on a different machine than SQL Server, you must enable TCP/IP and the right authentication mode.

  • Open SQL Server Configuration Manager
  • Go to SQL Server Network Configuration → Protocols for MSSQLSERVER
  • Enable TCP/IP → Right-click → Enable → Restart SQL Server service
  • In SSMS: Right-click Server → Properties → Security → Select "SQL Server and Windows Authentication mode"
  • Enable sa login: Security → Logins → sa → Properties → Status → Login: Enabled
⚠️ After changing authentication mode or enabling SA, you must restart the SQL Server service for changes to take effect.
